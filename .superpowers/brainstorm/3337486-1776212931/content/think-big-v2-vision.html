Patrol Mode (巡逻模式)

Agent 不需要你发消息才行动。配置 Patrol 后，它们会自主监控并在发现问题时主动通知你。

  • 🟢 PR Sentinel — 新 PR 自动 review，发现问题通知飞书
  • 🔴 Cost Watchdog — 每小时检查 AWS Cost Explorer，异常自动告警
  • 🟠 Dependency Auditor — 每日扫描 CVE，高危自动创建 PR 修复
  • 🔵 Infra Health — 定时检查 EKS/RDS/ALB 健康状态
# config.yaml
patrols:
  pr-review:
    trigger: "github:pr_opened"
    agent: code-reviewer
    prompt: "Review this PR for security issues"
    notify: feishu
  cost-alert:
    schedule: "@every 1h"
    agent: general
    prompt: "Check AWS cost anomalies vs yesterday"
    notify: [feishu, dashboard]
GAME CHANGER
🎯

Agent Workflows (可视化编排)

拖拽式 DAG 编排多个 Agent 组成工作流。前一个 Agent 的输出自动传递给下一个。

  • 示例 Pipeline: PR Opened → Code Review Agent → Security Scan Agent → If issues found → Fix Agent → Re-review → Auto Merge
  • 条件分支、并行执行、失败重试
  • Dashboard 实时可视化执行进度
  • Workflow 模板市场（社区共享）
👥

Team Agent Delegation

团队成员在飞书 @naozhi 时，Agent 先查你的知识库（Wiki + Obsidian）再回答，保持团队信息一致性。CTO 可以设置"代理回答"规则。

团队成员: @naozhi 我们的 WAF 误拦截率是多少？
Naozhi: [查询 Wiki "AWS WAF" 页面]
  "根据 4/11 的调优记录，当前误拦截率 0.4%
   (调整前 3.2%)。配置见 terraform-waf 模块。"
🔔

Proactive Insights (主动洞察)

基于你的知识图谱和对话历史，Agent 主动发现关联信息并推送。不是被动等你问。

  • "你昨天讨论的 Zeelool WAF 配置和上周 Giikin 遇到的 SizeRestrictions 问题相同，要不要统一修复？"
  • "检测到 terraform-waf 模块 3 天没更新，但 WAF Rule 优化的结论还未 apply"

AWS Console in Dashboard

不用切换到 AWS Console，在 Dashboard 里直接查询和操作基础设施。

Knowledge Graph Visualization

不只是搜索，是可视化的知识网络。看到项目、人、技术、决策之间的关系。

  • 力导向图：节点 = 实体（项目/服务/人/决策），边 = 引用关系
  • 时间轴模式：按时间展开决策链
  • 影响分析："如果我换掉 CloudFlare，哪些项目受影响？" → 高亮关联节点
  • 知识热力图：哪些领域知识丰富，哪些有盲区
  • 点击节点跳转到 Wiki 页面 / Obsidian 笔记 / Dashboard 对话
  • 社区检测（Louvain 算法）自动聚类知识领域
DIFFERENTIATOR
🕑

Decision Journal (决策日志)

自动从对话中提取架构决策，生成 ADR (Architecture Decision Records)。永远不再忘记"为什么当初这么做"。

  • Agent 识别对话中的决策模式："我们决定用 X 而不是 Y，因为..."
  • 自动生成 ADR: Context → Decision → Consequences
  • 写入 Obsidian vault（可配置路径）
  • 后续对话自动引用相关决策："这个方案和 ADR-042 冲突"
DIFFERENTIATOR
🚀

Session Replay & Sharing

完整回放一次问题解决过程，像看录像一样。可以分享给团队成员学习。

  • 带时间轴的回放：看到 Agent 的思考过程、工具调用、决策点
  • 可标注：在关键节点加注释（"这步是关键"）
  • 生成链接分享（token-based auth）
  • 用于团队 onboarding："看看 CTO 是怎么 debug 这个问题的"

Mobile Command Center

不是"能用"的移动端，是"想用"的移动端。

  • Quick Actions Widget — iOS/Android 桌面 widget，一键触发常用 Patrol
  • Voice-First — 长按说话，Agent 执行，语音播报结果

Session Teleport

借鉴 Claude Code 的 --teleport 机制，在设备间无缝转移对话上下文。

  • 手机上和 Agent 讨论方案 → 到电脑前 "teleport" → 继续写代码
  • CLI 里的 session → Dashboard 一键接管 → 飞书同步摘要
  • 反向：Dashboard session → push to CLI terminal

CTO Digital Twin

基于你所有的对话历史、决策记录、Obsidian 笔记，训练一个 "你的 Agent"。当你不在时，它可以代表你的决策风格和知识回答团队问题。

  • 团队成员问技术问题 → 先查 CTO Wiki → 用 CTO 的语气和判断风格回答
  • 不确定时标记 "需要 Keith 确认" 而不是乱答
  • 每周 Lint: 对比 Digital Twin 的回答和 CTO 的实际决策，持续校准

Predictive Architecture

基于项目历史和行业模式，预测架构瓶颈。

  • "按当前 DAU 增长趋势，3 个月后 RDS 会达到 IOPS 上限"
  • "Zeelool 的 CDN 配置和你处理 Giikin 时遇到的模式一致，建议提前..."
  • Weekly Architecture Review 自动生成并发送
MOONSHOT
🎯

Self-Improving System

Naozhi 观察你的使用模式，自动优化自己。

  • 检测到你每次 review PR 都会问同样的安全问题 → 自动加入 Patrol
  • 发现你经常搜索某类知识 → 自动触发 Ingest 编译相关 Wiki
  • 学习你的审批模式 → 低风险操作自动通过，只在高风险时暂停
  • 使用频率分析 → 自动调整 session TTL 和 max_procs
  • 检测到某个 Workflow 经常失败 → 建议优化或替换步骤
  • 周报: "本周 Naozhi 为你节省了约 X 小时"
